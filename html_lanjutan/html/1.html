Formulir Pendaftaran

Form dengan Method GET






Jika Anda mengeklik tombol "Kirim", data formulir akan dikirim ke halaman "/action_page.php".

Form dengan Method POST






Jika Anda mengeklik tombol "Kirim" pada form ini, data dikirim menggunakan metode POST dan ditampilkan di tab baru.

Kesimpulan

Pada uji coba pertama, ketika tombol "Reset" ditekan, semua input dalam formulir dikembalikan ke nilai default yang telah ditentukan sebelumnya. Sementara itu, ketika tombol "Kirim" ditekan dengan **method="GET"**, data formulir dikirim melalui URL dan dapat dilihat langsung di address bar. Hal ini menunjukkan bahwa metode GET tidak disarankan untuk data yang bersifat sensitif.

Ketika method diubah menjadi **POST**, data dikirim tanpa terlihat di URL, sehingga lebih aman untuk data penting. Pada uji coba atribut **target**, jika nilai `_self` digunakan, halaman saat ini akan diperbarui dengan hasil pengiriman formulir. Namun, jika `_blank` digunakan, hasil pengiriman formulir akan dibuka di tab baru. Hal ini bermanfaat jika ingin menampilkan data tanpa menutup halaman utama.

Secara keseluruhan, uji coba ini menunjukkan bahwa pemilihan metode (`GET` atau `POST`) serta atribut **target** mempengaruhi bagaimana data formulir dikirim dan ditampilkan. Pemahaman ini penting dalam mengembangkan aplikasi berbasis web yang aman dan sesuai dengan kebutuhan pengguna.
